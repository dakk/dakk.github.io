Working with opendata and using html5 as framework for your Android apps, will take you to an hard problem: how I can access data hosted in different origin? That's the Same-origin policy problem.

My solution for this problem, use a bi-directional communication between the Android java application and the javascript software, and could be used in different cases; the solution is divided in 3 part of software:

  • Android JSInterface that provide an function that prepare and execute a new task.
  • Android Async Task that start the download and will call a js callback when the download is finished.
  • Javascript part that call the JSInterface, and provide a callback for the finished download.
